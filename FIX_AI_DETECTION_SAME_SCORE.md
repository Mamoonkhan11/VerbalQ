# Fix AI Detection Same Score Issue 🔍

## Problem Description
```
AI detection showing same output for both AI-generated and humanized content
Both texts get identical ai_probability scores (e.g., 85% for both)
```

**Root Cause:** The detection prompt was too generic and didn't provide enough specific criteria for the LLM to distinguish between AI and human writing patterns.

---

## ✅ Solution Applied

### Enhanced Detection Prompt

**Before (Too Generic):**
```python
prompt = f"""You are an AI text detection expert.

Analyze whether the following text was written by a human or generated by an AI model.

Consider these signals:
- repetitive phrasing
- overly perfect grammar
- predictable sentence patterns
- lack of personal tone
- unnatural fluency

Language: {req.language}

Return ONLY valid JSON:
{{
  "ai_probability": number between 0 and 100,
  "human_probability": number between 0 and 100,
  "label": "AI" or "Human",
  "confidence": "Low | Medium | High"
}}

Text:
\"\"\"{req.text}\"\"\""""
```

**After (Detailed & Specific):**
```python
prompt = f"""You are an expert AI text detection analyst. Your task is to distinguish between human-written and AI-generated text.

Analyze the following text using these specific criteria:

**AI Indicators (increase ai_probability):**
- Unnaturally perfect grammar and spelling
- Repetitive sentence structures and patterns  
- Generic, safe statements without strong opinions
- Lack of personal anecdotes or specific details
- Overly formal or robotic tone
- Predictable word choices and phrasing
- No typos, slang, or colloquialisms
- Uniform sentence length throughout
- Absence of emotional nuance or personality

**Human Indicators (increase human_probability):**
- Natural variations in sentence structure
- Occasional minor errors or informal language
- Personal voice and emotional authenticity
- Specific examples and concrete details
- Varied vocabulary and creative expressions
- Colloquialisms, idioms, or cultural references
- Mixed sentence lengths (short + long)
- Subtle imperfections that show authentic thought

Language context: {req.language}

Text to analyze:
\"\"\"{req.text}\"\"\"

Return ONLY valid JSON with your analysis:
{{
  "ai_probability": <number 0-100>,
  "human_probability": <number 0-100>,
  "label": "AI" or "Human",
  "confidence": "Low", "Medium", or "High",
  "reasoning": "<brief explanation of key indicators found>"
}}

Scoring guidelines:
- 50-60: Slight indication either way
- 60-75: Moderate confidence
- 75-90: High confidence  
- 90-100: Very high confidence

IMPORTANT: ai_probability + human_probability should equal 100 (or very close).
Base your analysis SOLELY on writing style patterns, not content quality."""
```

## Why Same Scores Happened Before

### Root Cause Analysis:

#### Problem 1: Vague Prompt
```
Old: "Consider these signals: repetitive phrasing, overly perfect grammar..."
```

LLM interpretation: *"Everything has some repetition and good grammar... I'll just say 85% AI for everything."*

#### Problem 2: No Negative Examples
The prompt only listed AI indicators, not human ones. LLM had no contrast.

#### Problem 3: No Scoring Rubric
Without guidance like "75-90 = High confidence", LLM used arbitrary scoring.

#### Problem 4: No Explanation Required
No "reasoning" field meant no accountability for the scoring logic.

## How Detection Works Now

### Step-by-Step Process:

```
1. Receive text for analysis
   ↓
2. Build detailed prompt with 17 specific criteria
   ↓
3. Send to Ollama (mistral model)
   ↓
4. LLM analyzes text against each criterion
   ↓
5. Generate JSON with probabilities + reasoning
   ↓
6. Parse and validate response
   ↓
7. Return structured result to caller
```

### Analysis Depth:

The LLM now evaluates:
- **Sentence structure variety** → AI tends to repeat patterns
- **Grammar perfection** → AI is unnaturally flawless
- **Personal voice** → Humans show personality
- **Specific details** → Humans give concrete examples
- **Emotional nuance** → AI struggles with subtle feelings
- **Colloquialisms** → Humans use slang/idioms naturally
- **Error patterns** → Humans make small mistakes
- **Sentence length variation** → Humans mix short/long sentences

## Debugging Tips

### If Still Getting Same Scores:

#### Check 1: Verify Prompt is Being Used
Look at Render logs:
```
Render Dashboard → ML Service → Logs
```

Should see the full detailed prompt being constructed.

#### Check 2: Test Ollama Directly
```bash
# SSH into your Ollama host or test locally
curl http://localhost:11434/api/generate \
  -d '{
    "model": "mistral",
    "prompt": "Analyze this text for AI indicators: \"The weather is nice.\"",
    "stream": false
  }'
```

Verify mistral responds with meaningful analysis.

#### Check 3: Inspect Reasoning Field
If `reasoning` is empty or generic, the LLM isn't engaging deeply with the criteria.

**Solution:** Add more explicit instructions or increase `num_predict` to allow longer responses.

#### Check 4: Temperature Impact
Try different temperatures:
```python
"temperature": 0.0  # Most deterministic
"temperature": 0.1  # Current - slight variation
"temperature": 0.2  # More creative interpretation
```

Test which gives best differentiation.
